→ Configure SFTP connection to receive daily exports from Intuitive Health database
→ Build data mapping from source CSV/export to HubSpot Patient Visits object
→ Configure daily sync schedule to run after JV partner data loads (morning/early afternoon)
→ Configure batch upsert logic for ~2,000 daily records
→ Document export file format requirements and SFTP credentials setup 		→ Intuitive Health provides daily CSV/Excel exports via SFTP
→ Daily volume approximately 2,000 records (80% from JV partners, 20% owned facilities)
→ Data arrives in batches morning/early afternoon from various sources
→ Andrew's team prepares consolidated data table from 3 SQL sources
→ No direct SQL or API connections to Intuitive Health database
→ Skyvia or comparable SFTP integration platform
→ Daily sync frequency is sufficient for marketing automation needs
→ Includes up to 3 iterations of mapping refinement
3 20 Contact Association & Deduplication Workflows Intuitive Health wants to identify when the same patient visits multiple brands/locations and ensure marketing communications are properly coordinated while maintaining accurate patient visit history → Build workflow to associate Patient Visits to Contacts using email as primary match
→ Configure secondary matching using MRN (Medical Record Number) for identification
→ Create deduplication logic using compound matching (First Name + Last Name + Address + DOB)
→ Build "Most Recent Visit" stamp on contact records for segmentation 		→ Email is primary matching field when available
→ MRN serves as secondary identifier but varies by health system
→ Compound matching uses: First Name, Last Name, Address, Date of Birth
→ Some patients may legitimately visit multiple brands (vacation/travel scenarios)
→ Deduplication preserves visit history across all locations
4 15 Patient Segmentation & Marketing Automation Foundation Intuitive Health wants to enable targeted marketing journeys based on patient visit type, location, demographics, and visit history to build relationships beyond transactional care → Create active lists for key patient segments (Urgent Care patients, ER patients)
→ Build geographic segmentation by facility location and brand
→ Set up NPS score-based lists (promoters, passives, detractors)
→ Create visit recency lists (recent patients, 6-month inactive, 18-month inactive) 		→ Priority journeys: Urgent Care to ER cross-sell, geographic health alerts, re-engagement
→ Secondary use cases: new facility openings, seasonal health education, holiday hours
