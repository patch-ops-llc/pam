Implementation Methodology: Custom Event Packages

Event Package Structure

Each custom event is delivered as a complete, integrated package including:

  • Custom Event Configuration: JavaScript-based event setup using HubSpot's global tracking script
  • Lead/Engagement Scoring Updates: Event factored into scoring models with appropriate weights
  • Workflow Automation: Targeted workflows triggered by event occurrence (nurture, intervention, sales alerts)
  • Dashboard Integration: Event data incorporated into relevant reporting dashboards
  • Documentation: Event purpose, trigger criteria, scoring impact, and workflow logic documented

Agile Delivery: Event packages delivered iteratively in groups of 2-3, allowing for testing, feedback, and refinement before expanding to full implementation. This approach ensures maximum business impact and provides a proven framework for future event expansion.

Discovery-Driven Event Selection

Rather than prescribing specific events, we'll work collaboratively during discovery to identify the highest-value custom events based on:

  • Conversion Hypotheses: Events that correlate with trial-to-paid conversion
  • Upgrade Intent Signals: Behaviors indicating readiness to move to higher-tier packages
  • Engagement Quality: Actions that demonstrate real product value realization
  • Churn Risk Indicators: Behaviors or lack thereof that predict trial abandonment
  • Enterprise Signals: Usage patterns that indicate enterprise-level needs

This discovery-driven approach ensures we focus on events with proven business impact rather than assumptions about user behavior.
